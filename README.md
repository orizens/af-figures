# Intro

A pure client-side React SPA that displays and filters app reviews from the Appfigures API.

Built with **Bun · Vite 8 · React 19 · TanStack Router v1 · TanStack Query v5 · Tailwind CSS v4 · TypeScript**.

## Setup

### Prerequisites

| Tool                  | Version | Install                                     |
| --------------------- | ------- | ------------------------------------------- |
| [Bun](https://bun.sh) | ≥ 1.1   | `curl -fsSL https://bun.sh/install \| bash` |

> **macOS with Homebrew?**
>
> - Bun: `brew install oven-sh/bun/bun` or `npm i -g bun`

Either runtime works for running the app and E2E tests. **Bun is additionally required for unit tests** (the unit test script calls `bun test` directly).

### Install dependencies

```bash
# with Bun
bun install
```

### Install Playwright browsers (for E2E tests)

```bash
# with Bun
bunx playwright install chromium
```

## Running the app

### Development server

```bash
# with Bun
bun run dev
```

Opens at [http://localhost:5173](http://localhost:5173). The route tree (`src/routeTree.gen.ts`) is auto-generated by TanStack Router on the first run.

### Production build

```bash
# with Bun
bun run build
```

Output goes to `dist/`. Preview the build locally:

```bash
# with Bun
bun run preview
```

## Testing

### Unit tests

Unit tests use **Bun's built-in test runner**. Bun must be installed for this step — the test script calls `bun test` directly and there is no npm equivalent without adding a separate test framework.

```bash
bun run test:unit
```

This runs all `*.test.ts` files under `src/`. Tests cover pure utilities like date grouping logic.

Run in watch mode during development:

```bash
bun test --watch src/
```

---

### E2E tests (Playwright)

#### 1. Install Playwright

`@playwright/test` is already listed as a dev dependency, so `bun install` / `npm install` takes care of the package itself.
You still need to download the actual browser binaries once:

```bash
# with Bun
bunx playwright install chromium
```

To also install Firefox and WebKit (optional):

```bash
bunx playwright install   # or: npx playwright install
```

If you're on a fresh Linux/CI machine, install the OS-level dependencies too:

```bash
bunx playwright install --with-deps chromium
# or: npx playwright install --with-deps chromium
```

#### 2. Run all E2E tests

Playwright starts the dev server automatically (on port 5173) — **do not start it manually** beforehand.

```bash
# with Bun
bun run test:e2e
```

#### 3. Useful Playwright flags

| Goal                             | Bun                                              |
| -------------------------------- | ------------------------------------------------ |
| Run a single spec                | `bunx playwright test tests/e2e/reviews.spec.ts` |
| Run one test by name             | `bunx playwright test -g "shows reviews list"`   |
| Run in headed mode (see browser) | `bunx playwright test --headed`                  |
| Open interactive UI mode         | `bunx playwright test --ui`                      |
| Show HTML report after a run     | `bunx playwright show-report`                    |
| Debug step-by-step               | `bunx playwright test --debug`                   |

#### 4. Test output

After a run, Playwright saves an HTML report to `playwright-report/`. Open it with:

```bash
bunx playwright show-report   # or: npx playwright show-report
```

Failed test traces and screenshots are saved under `test-results/`.

## Tech choices at a glance

- **URL as sole app state** — search query, rating filter, and page are stored in the URL so links are always shareable.
- **TanStack Router** with Zod `validateSearch` keeps search params type-safe and validated.
- **TanStack Query** with `useInfiniteQuery` handles pagination and caching; `placeholderData` keeps the previous page visible while the next loads.
- **Accessibility-first** — semantic landmarks (`<search>`, `<main>`, `role="feed"`), `aria-live` regions, keyboard-navigable filters.
